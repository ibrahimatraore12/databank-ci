# dataBank CI — Customer 360

> *[Version française disponible : [README.md](README.md)]*

End-to-end analytics engineering platform for dataBank CI: ingestion, data
quality, dbt transformation, customer disengagement scoring, Streamlit
dashboard and an MCP server for natural-language exploration.

**Author:** Ibrahima TRAORÉ — Analytics Engineer
**Stack:** Python · pyenv · dbt · DuckDB · MLflow · Streamlit · Docker

## Architecture

```
starter_dataset.xlsx (10 tables)
        │
        ▼
   src/ingest.py ──► Bronze (DuckDB, real + synthetic)
        │
        ▼
   dbt_project/models/staging      (10 models, typing + business corrections)
        │
        ▼
   dbt_project/models/intermediate (behavioral computations)
        │
        ▼
   dbt_project/models/marts        (Gold: customer_360, customer_segments, nba)
        │
        ├──► dashboard/  (Streamlit, 8 pages, FR/EN)
        ├──► mcp_server/ (MCP server, 5 read-only tools)
        └──► ml/         (rule-based score + compared ML models)
```

## Installation

```bash
pyenv virtualenv 3.11.9 databank-ci-env
cd databank-ci
pyenv local databank-ci-env
pip install -r requirements.txt   # or see the package list below
cp .env.example .env
```

## Running the full pipeline

```bash
# 1. Bronze + enrichment + synthetic generation
python3 pipelines/run_pipeline.py

# 2. dbt transformation (Bronze -> Silver -> Gold)
cd dbt_project
export DBT_PROFILES_DIR=$(pwd)
dbt run
dbt test
cd ..

# 3. ML pipeline (rule-based score + model comparison + MLflow)
python3 pipelines/run_ml_pipeline.py

# 4. Dashboard
streamlit run dashboard/APP.py

# 5. MCP server
python3 mcp_server/databank_mcp_server.py
```

## Code rules

- No object-oriented programming in business logic: functions only.
- Bilingual comments (French then English) on non-obvious points.
- Strict semantic layer: no technical column name in the dashboard, see
  `dashboard/components/ui.py::LABELS` and `dashboard/i18n/`.
- Idempotence: `pipelines/run_pipeline.py` cleans its outputs before
  recreating them; `seed=42` fixed everywhere (dbt, ML, synthetic generation).
- All generated data is flagged `is_synthetic=True` and never mixed with
  real data in the default views.
