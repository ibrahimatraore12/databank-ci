# Data Dictionary — dataBank CI Customer 360

> *[French version: [data_dictionary.md](data_dictionary.md)]*

**Author:** Ibrahima TRAORÉ — Analytics Engineer
**Date:** July 2026

This dictionary covers the 3 Gold-layer tables (`dbt_project/models/marts/`)
consumed by the dashboard, the MCP server, and the ML pipeline. For the
source table schema, see `docs/erd_diagram_en.md`.

## 1. `customer_360` — one row per customer

| Column | Type | Description |
|--------|------|--------------|
| `customer_id` | varchar (PK) | Unique customer identifier |
| `full_name` | varchar | Full name |
| `segment` | varchar | `Mass` / `Affluent` / `Premier` / `Youth` |
| `risk_band` | varchar | `Low` / `Medium` / `High` — declared credit risk level |
| `city`, `district` | varchar | Location |
| `monthly_income_xof` | decimal | Declared monthly income (FCFA) |
| `preferred_channel` | varchar | Customer-declared preferred banking channel |
| `mobile_app_active`, `internet_banking_active`, `mobile_money_linked` | boolean | Activation of the 3 digital channels |
| `salaire_domicilie` | boolean | Salary domiciled, corrected from observed transactions (see `stg_accounts.sql`) |
| `recency_jours` | integer | Days since the last transaction (999 if no transaction observed) |
| `tendance_transactions` | integer | Transaction count in the last 30 days minus the 30 days before that |
| `nb_txn_30j`, `nb_txn_90j` | integer | Transaction count over the 30-day / 90-day windows |
| `tendance_3m` | integer | Transaction count in the last 90 days minus the 90 days before that |
| `nb_reclamations_ouvertes`, `nb_reclamations_total`, `nb_reclamations_severite_haute` | integer | Complaint aggregates |
| `score_digital` | integer (0-3) | Number of digital channels activated |
| `nb_comptes`, `nb_cartes`, `nb_produits_total` | integer | Number of products held |
| `solde_total_xof` | decimal | Sum of current balances across all of the customer's accounts |
| `avg_balance_90d_xof` | decimal | Sum of 90-day average balances across all accounts |
| `nbi_estime_xof` | decimal | Estimated NBI (standard UEMOA formula, **not** the customer's real accounting NBI — see `docs/decisions.md`) |
| `canal_majoritaire` | varchar | Customer's most-used channel, by transaction count |
| `dpd_max` | integer | Maximum payment delinquency observed (days), across all loans |
| `anciennete_jours` | integer | Days since `onboarding_date` |
| `risque_composite` | decimal (0-100) | Disengagement risk score — weighted sum of 4 sub-scores (recency 40%, complaints 30%, digital 20%, trend 10%) |
| `is_high_value_at_risk` | boolean | Premier/Affluent segment AND recency > 60 days |
| `is_digitally_dormant_salary` | boolean | Salary domiciled AND digital score ≤ 1 |
| `is_complaints_churn_risk` | boolean | Open complaint AND recency > 60 days |
| `is_cross_sell_target` | boolean | Customer with no card |
| `is_salary_upsell_opportunity` | boolean | High income (≥ 500,000 FCFA) AND salary not domiciled |
| `is_synthetic` | boolean | `True` if the row was generated by business bootstrap — never shown to the end user |
| `updated_at` | timestamp | Timestamp of the last `dbt run` |

## 2. `customer_segments` — one row per segment

| Column | Type | Description |
|--------|------|--------------|
| `segment` | varchar (PK) | `Mass` / `Affluent` / `Premier` / `Youth` |
| `nb_clients` | integer | Number of customers in the segment |
| `risque_composite_moyen` | decimal | Average risk score for the segment |
| `nb_high_value_at_risk`, `nb_digitally_dormant_salary`, `nb_complaints_churn_risk`, `nb_cross_sell_target`, `nb_salary_upsell_opportunity` | integer | Counts of the 5 ontology classes per segment |
| `taux_salaire_domicilie_pct` | decimal | Share of customers with salary domiciled |
| `score_digital_moyen` | decimal | Average digital score for the segment |
| `updated_at` | timestamp | Timestamp of the last run |

## 3. `nba` — one row per customer (Next Best Action)

| Column | Type | Description |
|--------|------|--------------|
| `customer_id` | varchar (PK, FK → `customer_360`) | Customer identifier |
| `segment` | varchar | Customer's segment |
| `next_best_action` | varchar | Recommended commercial action (business rule, see `ml/rules.py`) |
| `risque_composite` | decimal | Customer's risk score, duplicated from `customer_360` to avoid a join on the dashboard side |
| `updated_at` | timestamp | Timestamp of the last run |

## 4. Semantic layer — from technical column to displayed label

None of the columns above appear as-is in the dashboard: every technical
name is translated by `dashboard/components/ui.py::LABELS`, then resolved in
the active language via `dashboard/i18n/{fr,en}.json`. The full mapping is in
`LABELS`; see also `docs/decisions.md` for the decision that enforces this
rule.

## 5. Environment variables

| Variable | Default if unset | Used by | Description |
|----------|--------------------|--------------|--------------|
| `ADMIN_PASSWORD` | `databank-admin` | `dashboard/pages/99_Administration.py` | Password for the Administration area |
| `MCP_SERVER_URL` | none (required) | `dashboard/components/mcp_client.py` | HTTP URL of the deployed MCP server (e.g. `https://databank-ci-mcp-….run.app/mcp`) |
| `MCP_API_KEY` | none (auth disabled if unset) | `mcp_server/databank_mcp_server.py`, `dashboard/components/mcp_client.py` | Shared key required in the `X-API-Key` header on the MCP server's HTTP transport |
| `MCP_TRANSPORT` | `stdio` | `mcp_server/databank_mcp_server.py` | `stdio` locally, `streamable-http` in production |
| `PORT` | `8080` | `mcp_server/databank_mcp_server.py`, Dockerfile | Listening port (injected automatically by Cloud Run) |
| `GCS_BUCKET_NAME` | none (persistence disabled if unset) | `src/storage_sync.py` | GCS bucket for data persistence beyond a Cloud Run instance's lifecycle — see `docs/architecture_en.md` section 6 |
