# Data Dictionary - dataBank CI Customer 360

> *[French version: [data_dictionary.md](data_dictionary.md)]*

**Author:** Ibrahima TRAORÉ - Analytics Engineer
**Date:** July 2026

This document describes the 3 final tables (Gold layer,
`dbt_project/models/marts/`), the ones used by the dashboard, the MCP
server, and the Machine Learning pipeline. To see how the source tables
relate to each other, check `docs/erd_diagram_en.md`.

Reading note: `PK` stands for "primary key" (the unique identifier of each
row). `FK` stands for "foreign key" (a column that points to the
identifier of another table).

## 1. `customer_360` - one row per customer

| Column | Type | Description |
|--------|------|--------------|
| `customer_id` | varchar (PK) | Unique customer identifier |
| `full_name` | varchar | Full name |
| `segment` | varchar | `Mass` / `Affluent` / `Premier` / `Youth` |
| `risk_band` | varchar | `Low` / `Medium` / `High` - declared credit risk level |
| `city`, `district` | varchar | Customer location |
| `monthly_income_xof` | decimal | Declared monthly income, in FCFA |
| `preferred_channel` | varchar | Customer-declared preferred banking channel |
| `mobile_app_active`, `internet_banking_active`, `mobile_money_linked` | boolean | Whether each of the 3 digital channels is activated |
| `salaire_domicilie` | boolean | Whether the salary is domiciled at the bank, checked against actually observed transactions (see `stg_accounts.sql`) |
| `recency_jours` | integer | Number of days since the last transaction (999 if no transaction observed) |
| `tendance_transactions` | integer | Number of transactions in the last 30 days, minus those from the 30 days before that |
| `nb_txn_30j`, `nb_txn_90j` | integer | Number of transactions over the last 30 / 90 days |
| `tendance_3m` | integer | Number of transactions in the last 90 days, minus those from the 90 days before that |
| `nb_reclamations_ouvertes`, `nb_reclamations_total`, `nb_reclamations_severite_haute` | integer | Complaint counts |
| `score_digital` | integer (0 to 3) | Number of digital channels activated |
| `nb_comptes`, `nb_cartes`, `nb_produits_total` | integer | Number of products held by the customer |
| `solde_total_xof` | decimal | Sum of current balances across all of the customer's accounts |
| `avg_balance_90d_xof` | decimal | Sum of 90-day average balances across all accounts |
| `nbi_estime_xof` | decimal | Revenue generated by the customer (NBI), estimated with the standard WAEMU formula. This is **not** the customer's real accounting figure - see `docs/decisions_en.md` |
| `canal_majoritaire` | varchar | Customer's most-used channel, by transaction count |
| `dpd_max` | integer | Largest payment delay observed (in days), across all loans |
| `anciennete_jours` | integer | Number of days since the customer's start date (`onboarding_date`) |
| `risque_composite` | decimal (0 to 100) | Disengagement risk score: weighted sum of 4 sub-scores (inactivity 40%, complaints 30%, digital usage 20%, trend 10%) |
| `is_high_value_at_risk` | boolean | True if the customer is in the Premier or Affluent segment AND inactive for more than 60 days |
| `is_digitally_dormant_salary` | boolean | True if the salary is domiciled AND the digital score is 1 or lower |
| `is_complaints_churn_risk` | boolean | True if the customer has an open complaint AND has been inactive for more than 60 days |
| `is_cross_sell_target` | boolean | True if the customer has no card |
| `is_salary_upsell_opportunity` | boolean | True if income is high (500,000 FCFA or more) AND the salary is not domiciled |
| `is_synthetic` | boolean | `True` if the row was artificially generated. Never shown to the end user |
| `updated_at` | timestamp | Date and time of the last `dbt run` |

## 2. `customer_segments` - one row per segment

| Column | Type | Description |
|--------|------|--------------|
| `segment` | varchar (PK) | `Mass` / `Affluent` / `Premier` / `Youth` |
| `nb_clients` | integer | Number of customers in the segment |
| `risque_composite_moyen` | decimal | Average risk score for the segment |
| `nb_high_value_at_risk`, `nb_digitally_dormant_salary`, `nb_complaints_churn_risk`, `nb_cross_sell_target`, `nb_salary_upsell_opportunity` | integer | Number of customers affected by each of the 5 categories, for this segment |
| `taux_salaire_domicilie_pct` | decimal | Share of customers with a domiciled salary |
| `score_digital_moyen` | decimal | Average digital score for the segment |
| `updated_at` | timestamp | Date and time of the last run |

## 3. `nba` - one row per customer (Next Best Action)

| Column | Type | Description |
|--------|------|--------------|
| `customer_id` | varchar (PK, FK → `customer_360`) | Customer identifier |
| `segment` | varchar | Customer's segment |
| `next_best_action` | varchar | Recommended commercial action for this customer (computed by a business rule, see `ml/rules.py`) |
| `risque_composite` | decimal | Customer's risk score, copied from `customer_360` to avoid an extra calculation on the dashboard side |
| `updated_at` | timestamp | Date and time of the last run |

## 4. How a technical column name becomes displayed text

None of the columns above appear as-is on the dashboard. Every technical
name is first translated into readable text by
`dashboard/components/ui.py::LABELS`, then displayed in the right language
using the `dashboard/i18n/{fr,en}.json` files. The full list of these
mappings is in `LABELS`. See also `docs/decisions_en.md` to understand why
this rule was put in place.

## 5. Environment variables (configuration settings)

| Variable | Default if unset | Used by | Description |
|----------|--------------------|--------------|--------------|
| `ADMIN_PASSWORD` | `databank-admin` | `dashboard/pages/99_Administration.py` | Password to access the Administration area |
| `MCP_SERVER_URL` | none (required) | `dashboard/components/mcp_client.py` | Address of the live MCP server (example: `https://databank-ci-mcp-….run.app/mcp`) |
| `MCP_API_KEY` | none (auth check disabled if unset) | `mcp_server/databank_mcp_server.py`, `dashboard/components/mcp_client.py` | Secret key required to connect to the live MCP server |
| `MCP_TRANSPORT` | `stdio` | `mcp_server/databank_mcp_server.py` | Communication mode: `stdio` locally, `streamable-http` in production |
| `PORT` | `8080` | `mcp_server/databank_mcp_server.py`, Dockerfile | App's listening port (set automatically by Cloud Run) |
| `GCS_BUCKET_NAME` | none (backup disabled if unset) | `src/storage_sync.py` | Google Cloud Storage space used to keep data beyond a Cloud Run instance's lifetime - see `docs/architecture_en.md`, section 6 |
